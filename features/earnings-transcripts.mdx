## Overview

Earnings Transcripts is Finance Agent's core feature for analyzing quarterly earnings calls from 2020-2025. Get word-for-word executive commentary, financial metrics, and forward guidance directly from the source that professional analysts use.

Unlike generic LLMs that summarize web content, Finance Agent retrieves actual transcript chunks from your target companies and quarters, giving you precise answers with exact citations.

## How It Works

When you ask a question about earnings calls, Finance Agent:

1. **Plans the Search** - Generates focused keyword phrases optimized for semantic search (not verbose questions)
2. **Parallel Retrieval** - Searches all target quarters simultaneously using hybrid vector + keyword search
3. **Answer Generation** - Synthesizes findings with precise [TC-N] citation markers
4. **Quality Evaluation** - Assesses completeness and iterates if gaps are found (up to 2 iterations)

## Features

### Hybrid Search
Combines vector embeddings (70%) with keyword matching (30%) for optimal recall:
- **Vector search**: Captures semantic similarity ("revenue" matches "sales", "income")
- **Keyword search**: Ensures exact terms are prioritized ("billings" won't miss "billings")

### Planning-Driven Retrieval
Before searching, the agent generates:
- **Sub-questions**: What specific data is needed?
- **Search queries**: Dense keyword phrases optimized for embedding search

Example:
```
User question: "What is Datadog's revenue per employee?"

Agent plan:
  Sub-questions:
    - What was Datadog's total revenue?
    - How many employees does Datadog have?
  
  Search queries:
    - "total revenue subscription professional services"
    - "number of employees workforce headcount"
```

### Citation Format
Every answer includes **[TC-N]** markers that link to source transcripts:

- **[TC-1]** = First transcript chunk retrieved
- **[TC-5]** = Fifth transcript chunk
- Click any citation to see the full excerpt with company, quarter, and year metadata

### Multi-Ticker Synthesis
When comparing companies, Finance Agent:
1. Runs parallel searches for each ticker
2. Renumbers citations to avoid conflicts (AAPL gets TC-1 to TC-10, MSFT gets TC-11 to TC-20)
3. Synthesizes findings into a unified comparative answer

## Coverage

- **Time Range**: 2020-2025 (coverage varies by company)
- **Companies**: S&P 500 and major tech companies
- **Quarters**: Q1, Q2, Q3, Q4 (fiscal quarters, varies by company fiscal year)
- **Database**: PostgreSQL with pgvector for semantic search

<Note>
Quarter availability varies by company. Use specific quarter references ("Q3 2024") or relative terms ("last 3 quarters", "latest quarter") to get the most recent data available.
</Note>

## Technical Details

### Architecture
- **Chunk Size**: 1000 characters with 200-character overlap
- **Embedding Model**: all-MiniLM-L6-v2 (384 dimensions)
- **Chunks per Query**: Up to 15 per quarter, 20 shown to LLM per iteration
- **Max Iterations**: 2 (initial generation + 1 refinement)
- **Quality Threshold**: 85% confidence to stop early

### Retrieval Flow
```
User Question
  ↓
Plan Generation (sub-questions + search queries)
  ↓
Parallel Hybrid Search (vector + keyword across all target quarters)
  ↓
Answer Generation (with [TC-N] citations)
  ↓
Quality Evaluation
  ↓
[If gaps found] → Replan with additional queries → Search again
  ↓
Final Answer
```

## Limitations

- **No Real-Time Data**: Transcripts are from published earnings calls, typically 1-2 quarters behind current date
- **Fiscal Year Variations**: Companies define fiscal years differently (Apple's Q4 is Oct-Dec, Microsoft's Q4 is Jul-Sep)
- **Coverage Gaps**: Not all companies have complete historical data back to 2020
- **No Stock Prices**: Focus is on earnings call commentary, not market data
